# Literature Map

## 1. Mixed-Initiative Interaction and Automation

Purpose: build the theoretical base for shared control between human and intelligent systems.

Key references:

- Horvitz, E. (1999). Principles of Mixed-Initiative User Interfaces.
- Parasuraman, R., Sheridan, T. B., & Wickens, C. D. (2000). A Model for Types and Levels of Human Interaction with Automation.
- Amershi et al. (2019). Guidelines for Human-AI Interaction.

How this supports the project:

These works help explain why the core issue is not whether agents are powerful, but how initiative, control, correction, and intervention are distributed between humans and systems.

## 2. Supervisory Control and Human-Swarm Interaction

Purpose: borrow knowledge from fields where one human supervises multiple autonomous entities.

Key references:

- Chen, J. Y. C., Barnes, M. J., & Harper-Sciarini, M. (2011). Supervisory Control of Multiple Robots.
- Chen & Barnes (2014). Human-Agent Teaming for Multirobot Control.
- Kolling et al. (2013). Human-Swarm Interaction.
- Patel et al. (2019). Mixed-Granularity Human-Swarm Interaction.
- Hussein et al. (2022). Characterization of Indicators for Adaptive Human-Swarm Teaming.

How this supports the project:

These works support the idea that users need appropriate control granularity, situation awareness, and intervention mechanisms when managing multiple autonomous agents.

## 3. Multi-Agent LLM Systems

Purpose: show that multi-agent AI systems are technically emerging but still lack user-facing orchestration language.

Key references:

- AutoGen: Enabling Next-Gen LLM Applications via Multi-Agent Conversation.
- MetaGPT: Meta Programming for A Multi-Agent Collaborative Framework.
- More Agents Is All You Need.
- Towards a Science of Scaling Agent Systems.
- Single-Agent LLMs Outperform Multi-Agent Systems on Multi-Hop Reasoning Under Equal Thinking Token Budgets.

How this supports the project:

The project should avoid claiming multi-agent systems are always better. Instead, it should ask when and how users should command agent teams.

## 4. HCI Multi-Agent Prototype Precedents

Purpose: find nearby HCI works that use multiple agents as interaction material.

Key references:

- Social Simulacra.
- Generative Agents.
- Multi-Agent Strategy Explanations for Human-Robot Collaboration.
- CritiqueCrew: Orchestrating Multi-Perspective Conversational Design Critique.

How this supports the project:

These works show that agent plurality can be designed as an interface experience, but most examples use fixed roles or simulation. This project explores dynamic formations as a user-facing command method.
