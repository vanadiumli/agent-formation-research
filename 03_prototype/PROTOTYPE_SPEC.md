# Prototype Spec

## Prototype Name

Formation Command Interface

## Core User Scenario

A user needs to complete a complex digital task, such as preparing a research brief, comparing design directions, or planning a competition proposal. Instead of asking one AI assistant repeatedly, the user commands a small team of agents through formations.

## Core Screens

1. Formation Switcher

The user can switch between scouting, debate, execution, and audit formations.

2. Task Map

The system visualizes task decomposition, active branches, dependencies, and current progress.

3. Agent Unit Cards

Each agent is shown as a unit with role, capability, permission, current action, confidence/risk state, and latest output.

4. Situation Panel

The interface highlights conflicts, blocked tasks, high-risk actions, and moments that need user confirmation.

5. Replay and Intervention

The user can inspect how an output was produced, pause a formation, reroute a task, or take over a specific step.

## Minimal v1

For the first prototype, build only:

- Formation switcher.
- Task map.
- Agent unit cards.
- Replay/intervention panel.

The goal is not full automation. Wizard-of-Oz or simulated agent outputs are acceptable for the first study.
