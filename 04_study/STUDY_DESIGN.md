# User Study Design

## Study Goal

Evaluate whether formation-based interaction helps users understand and command multi-agent collaboration better than existing interface patterns.

## Conditions

1. Chat-based single agent interface.
2. Fixed-role multi-agent interface.
3. Formation-based command interface.

## Candidate Task

Participants complete a complex knowledge-work task, such as generating a research topic brief or comparing design concepts. The task should include exploration, comparison, execution, and verification stages.

## Measures

Quantitative:

- Task completion quality.
- Time on task.
- Situation awareness.
- Perceived control.
- Cognitive load.
- Trust calibration.
- Error recovery performance.

Qualitative:

- How participants understood agent collaboration.
- When they wanted to intervene.
- Whether formations felt understandable or unnecessary.
- Which formation names and structures made sense.

## Sample Size

For a master thesis pilot, start with 12-18 participants for prototype iteration. For a more complete comparison study, target 24-36 participants.

## Key Risk

The study must avoid becoming a superficial “game-like UI preference” test. The evaluation should focus on command, understanding, intervention, and recovery.
